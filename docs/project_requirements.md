# Parts-Master Inventory API Requirements

## Objective
Build a backend for a dealership inventory system that handles data cleaning, 
relational storage, and API access.

## Functionality
- **Data Ingestion:** Fetch automotive datasets via Kaggle API.
- **Wrangling:** Use Python/Pandas to merge parts with categories and standardize pricing.
- **Storage:** Use SQL Stored Procedures to manage stock levels and auto-restock triggers.
- **API:** Provide RESTful endpoints for part lookups and sales processing.
